# 云场景一：播放 → 暂停 → 继续播放 → 结束

> 本文档面向前后端联合协作，覆盖后端如何发送数据、前端如何接收处理、双方各自注意事项，以及当前实现进度。
> 服务地址：`http://localhost:8080`
> WebSocket 地址：`ws://localhost:8080/ws/planDeduce`

---

## 一、场景概述

用户以默认 1 倍速初始化播放，中途手动暂停一次，再继续播放，最终等待后端推送结束信号。

**完整交互顺序：**

```
前端                               后端
 |                                  |
 |-- 建立 WebSocket 连接 ---------->|  登记 sessionId -> WebSocketSession 映射
 |                                  |
 |-- GET /plan/sendPlanDeduce ----->|  创建/复用 ScenarioTask
 |                                  |  initializeRuntimeState()
 |                                  |  preloadFullSnapshots(dbName, 600)
 |                                  |  queryMaxSimTime(dbName) → 存入 maxSimTime
 |                                  |  running = true
 |                                  |  pushCurrentStateSnapshot("INIT")
 |                                  |  ensureWorkerScheduled() → 注册 1 秒 tick
 |<-- WS: INIT --------------------|
 |<-- WS: PLAY (t=1) -------------|  tick#1：running=true, speed=1, next=1
 |<-- WS: PLAY (t=2) -------------|  tick#2
 |        ...                       |
 |-- GET /plan/startOrStop?flag=0 ->|  task.pause()
 |                                  |  running = false
 |<-- WS: PAUSE -------------------|
 |                                  |  tick 继续触发但因 running=false 直接跳过
 |-- GET /plan/startOrStop?flag=1 ->|  task.resume()
 |                                  |  running = true
 |<-- WS: START -------------------|
 |<-- WS: PLAY (t=N+1) -----------|  tick 继续推进
 |        ...                       |
 |<-- WS: PLAY (t=maxSimTime) ----|  最后一帧
 |<-- WS: PAUSE -------------------|  tick 内触发结束逻辑
 |<-- WS: FINISH ------------------|
```

---

## 二、步骤详解

### 步骤 0：建立 WebSocket 连接

**前端操作：**
在调用任何 HTTP 接口之前，必须先建立 WebSocket 连接。

```javascript
const ws = new WebSocket('ws://localhost:8080/ws/planDeduce?sessionId=s1');
ws.onopen    = () => { /* 连接成功，允许发起 HTTP 控制命令 */ };
ws.onmessage = (e) => { handleMessage(JSON.parse(e.data)); };
ws.onclose   = () => { /* 处理断连逻辑 */ };
```

**后端处理（`PlanDeduceWebSocketHandler.afterConnectionEstablished`）：**
- 从 URL 参数解析 `sessionId`（缺省为 `"default"`）
- 将 `sessionId → WebSocketSession` 写入 `sessionMap`（ConcurrentHashMap）
- 若同一 `sessionId` 重连，新连接覆盖旧映射，确保推送始终命中最新连接

**注意：**
- WebSocket 未建立时，后端推送会被静默丢弃（`sendToSession` 中判断 session 是否存在且 open）
- 同一页面实例内 `sessionId` 必须固定，多标签页不要复用同一个 `sessionId`

### 步骤 1：初始化并开始播放

#### 1.1 前端发起请求

**请求方式：** GET
**请求路径：** `/plan/sendPlanDeduce`
**完整 URL：**
```
GET http://localhost:8080/plan/sendPlanDeduce?dbName=plandeduce&skip=0&sessionId=s1
```

**参数说明：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `dbName` | String | 是 | 业务库名，仅允许字母/数字/下划线 |
| `skip` | Integer | 否 | 从第几秒开始，默认 0 |
| `sessionId` | String | 否 | 会话标识，默认 `"default"` |

#### 1.2 后端处理链路

```
PlanDeduceController.sendPlanDeduce()
  ├─ taskManager.get(dbName, sessionId)         // 查找是否有现存任务
  ├─ 若存在且 isExecuting() == true              // running=true AND future 未取消
  │   └─ throw IllegalArgumentException          // → HTTP 400
  ├─ taskManager.getOrCreate(dbName, sessionId) // 创建或复用 ScenarioTask
  └─ task.start(skip=0, null)
       ├─ currentTime.set(0)                     // 时间游标归零
       ├─ initializeRuntimeState()
       │    ├─ progressDataService.preloadFullSnapshots(dbName, 600)
       │    │    └─ 构造并缓存 simTime=0 的全量快照
       │    ├─ maxSimTime.set(queryMaxSimTime(dbName))  // 查询库中最大业务秒
       │    └─ initialized.set(true)
       ├─ running.set(true)
       ├─ pushCurrentStateSnapshot("INIT")        // 推送 INIT 消息
       └─ ensureWorkerScheduled()                 // 注册周期性 tick（每 1000ms）
```

#### 1.3 HTTP 响应

- **状态码：** `200`
- **响应体：** 空

#### 1.5 后端推送 PLAY 消息（tick 驱动，每 1000ms 一次）

**tick 内部逻辑（`ScenarioTask.tickSafely`）：**
```
tickSafely()
  ├─ handlePendingSkip()          // 优先消费 skip 命令，有则本轮不推进播放
  ├─ if !running: return          // 暂停态直接跳过
  ├─ if speed <= 0: return        // 0 倍速等价于暂停
  ├─ if now >= maxSimTime         // 已到终点
  │   ├─ pause()                  // running=false
  │   └─ pushStatus("FINISH")    // 推 FINISH 后 return
  ├─ next = min(now + speed, maxSimTime)   // 计算本帧终点（speed=1 时 next=now+1）
  ├─ currentTime.set(next)
  └─ pushPlaySnapshot(now, next, speed)
       ├─ 若 next % 600 == 0：    // 全量间隔点（默认 600 秒）
       │   fullData = queryCachedFullData(dbName, 600, next)
       │   incrementalData = []
       └─ 否则：
           fullData = []
           incrementalData = queryIncrementalData(dbName, now, next)
           // 查询 (now, next] 区间内所有变化记录
```

**普通 PLAY 消息（非全量点，如 t=1）：**

#### 1.6 前端接收处理

```javascript
function handleMessage(msg) {
  switch (msg.type) {
    case 'INIT':
      state.initialized = true;
      state.running     = msg.running;    // true
      state.speed       = msg.speed;      // 1
      state.currentTime = msg.currentTime; // 0
      state.fullTime    = msg.fullTime;   // 0
      // 用 msg.data（或 msg.fullData）建立初始地图状态
      renderMap(msg.data);
      break;

    case 'PLAY':
      state.currentTime = msg.currentTime;
      state.fullTime    = msg.fullTime;
      state.speed       = msg.speed;
      state.running     = msg.running;
      // 刷新地图：直接消费 msg.data
      renderMap(msg.data);
      // 更新进度条
      updateProgressBar(msg.currentTime);
      break;
  }
}
```

**关键原则：**
- 进度条 `currentTime` 只能来自 `PLAY.currentTime`，禁止本地计时自行推进
- 收到 INIT 后才允许响应用户的后续操作（暂停、倍速等）

### 步骤 2：点击暂停

#### 2.1 前端发起请求

**请求方式：** GET
**请求路径：** `/plan/startOrStop`
**完整 URL：**
```
GET http://localhost:8080/plan/startOrStop?dbName=plandeduce&flag=0&sessionId=s1
```

**参数说明：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `dbName` | String | 是 | 业务库名 |
| `flag` | Integer | 是 | 0=暂停，1=开始 |
| `sessionId` | String | 否 | 会话标识 |

#### 2.2 后端处理链路

```
PlanDeduceController.startOrStop()
  └─ task.startOrStop(flag=0)
       └─ task.pause()
            ├─ running.set(false)
            ├─ speed.compareAndSet(0, 1)   // 若当前倍速为 0，兜底恢复为 1
            └─ pushStatus("PAUSE", calculateNearestFullTime(currentTime), "已暂停")
                  // calculateNearestFullTime: (currentTime / 600) * 600
```

#### 2.3 HTTP 响应

- **状态码：** `200`
- **响应体：** 空

#### 2.4 后端推送 PAUSE 消息

```json
{
  "type": "PAUSE",
  "dbName": "plandeduce",
  "sessionId": "s1",
  "currentTime": 5,
  "fullTime": 0,
  "speed": 1,
  "running": false,
  "message": "已暂停"
}
```

**注意：**
- PAUSE 消息**不携带** `fullData`/`incrementalData`/`data`（状态类消息，非快照类）
- `currentTime` 是暂停时刻的业务秒
- 收到 PAUSE 后，tick 仍然每 1000ms 触发一次，但因 `running=false` 在 tick 入口直接返回，不会推送任何数据

#### 2.5 前端接收处理

```javascript
case 'PAUSE':
  state.running     = false;      // msg.running = false
  state.currentTime = msg.currentTime;
  // 切换 UI 到暂停态（按钮变为"继续"）
  // 注意：不要在点击暂停按钮时立即切 UI，必须等收到 PAUSE 消息后再切
  updateUI('paused');
  break;
```

### 步骤 3：点击继续播放

#### 3.1 前端发起请求

**请求方式：** GET
**请求路径：** `/plan/startOrStop`
**完整 URL：**
```
GET http://localhost:8080/plan/startOrStop?dbName=plandeduce&flag=1&sessionId=s1
```

#### 3.2 后端处理链路

```
PlanDeduceController.startOrStop()
  └─ task.startOrStop(flag=1)
       └─ task.resume()
            ├─ if speed.get() <= 0: speed.set(1)    // 兜底倍速
            ├─ initialized.get() == true             // 已初始化，走恢复分支
            ├─ running.set(true)
            └─ pushStatus("START", calculateNearestFullTime(currentTime), "已开始")
                  // Worker 定时器不重新注册（ensureWorkerScheduled 幂等）
```

**与场景二的区别：**
此处 `initialized=true`（已经 start 过），走的是 `pushStatus("START")` 路径，而非重新推 INIT。

#### 3.3 HTTP 响应

- **状态码：** `200`
- **响应体：** 空

#### 3.4 后端推送消息

**先推 START：**

**随后 tick 恢复推送 PLAY（从暂停处接续）：**

```json
  ],

  "message": "当前时间 6 秒，返回第 6-6 秒增量数据"

```

#### 3.5 前端接收处理

```javascript

  // 注意：START 只表示"恢复命令已被接受"，真正时间推进仍以后续 PLAY 为准
  updateUI('playing');
  break;
```

### 步骤 4：等待播放自动结束

无需前端发起任何请求。后端 tick 推进到 `maxSimTime` 后自动触发结束逻辑。

#### 4.1 后端结束逻辑（`tickSafely` 内）

```
当 next >= maxSimTime 时（当前样例数据 maxSimTime=1200）：
  1. pushPlaySnapshot(now, maxSimTime, speed)   // 推最后一帧 PLAY（携带全量快照）
  2. pause()
       ├─ running.set(false)
       └─ pushStatus("PAUSE", ...)
  3. pushStatus("FINISH", calculateNearestFullTime(maxSimTime), "推演结束")
```

**最后一帧 PLAY（`currentTime=1200`，为全量间隔点）：**
```json
{
  "type": "PLAY",
  "dbName": "plandeduce",
  "sessionId": "s1",
  "currentTime": 1200,
  "fullTime": 1200,
  "speed": 1,
  "running": true,
  "fullData": [
    {
      "roomObjectId": 1,
      "currentPos": 100,
      "nextPos": 101,
      "direction": 240,
      "currentSpeed": 800,
      "visible": 1,
      "simTime": 1200,
      "targetId": 12001,
      "sourceType": "FULL"
    }
  ],
  "incrementalData": [],
  "data": [ { "simTime": 1200, "sourceType": "FULL", ... } ],
  "message": "当前时间 1200 秒，返回第 1200 秒全量数据"
}
```

**随后 PAUSE：**
```json
{
  "type": "PAUSE",
  "dbName": "plandeduce",
  "sessionId": "s1",
  "currentTime": 1200,
  "fullTime": 1200,
  "speed": 1,
  "running": false,
  "message": "已暂停"
}
```

**最后 FINISH：**
```json
{
  "type": "FINISH",
  "dbName": "plandeduce",
  "sessionId": "s1",
  "currentTime": 1200,
  "fullTime": 1200,
  "speed": 1,
  "running": false,
  "message": "推演结束"
}
```

#### 4.2 前端接收处理

```javascript
case 'FINISH':
  state.finished    = true;
  state.running     = false;
  state.currentTime = msg.currentTime;  // 1200
  // 禁用播放/暂停按钮，显示"推演结束"提示
  updateUI('finished');
  break;
```

**前端状态机建议：**
- 收到 `PAUSE` 时：仅停止时间推进（`running=false`），不判断是否结束
- 收到 `FINISH` 时：才真正标记 `finished=true`，触发结束 UI
- **禁止**仅凭 `currentTime >= maxSimTime` 自行判断结束

## 三、消息推送完整时序

```
t=0  → INIT  （初始化快照，fullData 有值）
t=1  → PLAY  （增量，incrementalData 有值）
t=2  → PLAY
...
t=5  → PLAY  → 用户点暂停 → HTTP 200
       → PAUSE（running=false）
       → tick 继续触发但不推送
       → 用户点继续 → HTTP 200
       → START（running=true）
t=6  → PLAY  （增量）
...
t=600 → PLAY （全量间隔点，fullData 有值，incrementalData 为空）
...
t=1200 → PLAY（最后一帧，全量间隔点）
        → PAUSE
        → FINISH
```

## 四、重要注意事项

### 前端

| 项目 | 说明 |
|------|------|
| WS 先于 HTTP | 必须先建立 WebSocket，再调用 `/sendPlanDeduce`，否则 INIT 消息会被丢弃 |
| UI 切换以 WS 为准 | 点击按钮后不要立即改 UI，等 PAUSE/START 消息到达后再切换 |
| 进度条来源 | `currentTime` 只认 `PLAY.currentTime`，禁止本地计时 |
| 结束判断 | 只认 `FINISH`，`PAUSE` 不代表结束 |
| `data` 字段 | 直接消费 `data`（= fullData + incrementalData 合并），无需自行拼接 |
| sessionId 固定 | 同一页面实例内不要改 sessionId，多标签页要用不同 sessionId |
| 重复初始化保护 | 任务执行中（`running=true`）再调 `sendPlanDeduce` 会返回 HTTP 400 |

### 后端

| 项目 | 说明 |
|------|------|
| Worker 不停止 | `pause()` 只让 tick 空转，不会真正取消 `ScheduledFuture`；只有 `destroy()` 才会 cancel |
| 全量点规则 | `next % fullSaveIntervalSeconds == 0` 时推全量，否则推增量；默认间隔 600 秒 |
| 增量数据语义 | PLAY 的 `incrementalData` 是 `(previousTime, nextTime]` 内的**全部**变化记录，区别于 INIT/SKIP 的"最后生效值" |
| 并发安全 | `tickSafely` 和 `start/skip/pause/resume` 均加 `synchronized`，WebSocket 推送用 `synchronized(session)` 保证有序 |
| speed=0 兜底 | `pause()` 内 `speed.compareAndSet(0, 1)` 避免恢复后仍处于 0 倍速假播放态 |
