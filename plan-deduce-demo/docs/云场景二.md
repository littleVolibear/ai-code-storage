# 云场景二：先设置 3 倍速 → 开始播放 → 暂停 → 继续播放 → 结束

> 本文档面向前后端联合协作，覆盖后端如何发送数据、前端如何接收处理、双方各自注意事项，以及当前实现进度。
> 服务地址：`http://localhost:8080`
> WebSocket 地址：`ws://localhost:8080/ws/planDeduce`

---

## 一、场景概述

页面进入时，用户希望以 3 倍速播放。前端先调用倍速接口预设倍速，再触发开始播放。后端在 `startOrStop(flag=1)` 时发现任务尚未初始化，自动完成初始化并以 3 倍速推进。

**场景二与场景一的核心区别：**
- 场景一：先调 `sendPlanDeduce` 初始化，`start()` 负责推 INIT
- 场景二：先调 `speed=3` 预设，再调 `startOrStop(flag=1)`，`resume()` 内部检测到 `initialized=false` 后自动完成初始化并推 INIT，**首次开始收到的是 INIT 而非 START**

**完整交互顺序：**

```
前端                               后端
 |                                  |
 |-- 建立 WebSocket 连接 ---------->|  登记 sessionId 映射
 |                                  |
 |-- GET /plan/speed?speed=3 ------>|  taskManager.getOrCreate(dbName, sessionId)
 |                                  |  task.setSpeed(3)
 |                                  |  speed = 3，running 不变（false）
 |<-- WS: SPEED -------------------|
 |                                  |
 |-- GET /plan/startOrStop?flag=1 ->|  task.resume()
 |                                  |  initialized == false → initializeRuntimeState()
 |                                  |  running = true
 |                                  |  pushCurrentStateSnapshot("INIT")
 |                                  |  ensureWorkerScheduled()
 |<-- WS: INIT -------------------|  speed=3，running=true
 |<-- WS: PLAY (t=3) -------------|  tick#1：step=3，next=3
 |<-- WS: PLAY (t=6) -------------|  tick#2
 |        ...                       |
 |-- GET /plan/startOrStop?flag=0 ->|  task.pause()
 |<-- WS: PAUSE ------------------|
 |                                  |
 |-- GET /plan/startOrStop?flag=1 ->|  task.resume()，initialized==true
 |                                  |  pushStatus("START")  ← 这次发 START，不发 INIT
 |<-- WS: START ------------------|
 |<-- WS: PLAY (t=N+3) -----------|
 |        ...                       |
 |<-- WS: PLAY (t=1200) ----------|
 |<-- WS: PAUSE ------------------|
 |<-- WS: FINISH ------------------|
```

---

## 二、步骤详解

### 步骤 0：建立 WebSocket 连接

与云场景一步骤 0 完全相同。
```javascript
const ws = new WebSocket('ws://localhost:8080/ws/planDeduce?sessionId=s2');
```

**关键点：** 必须先建连，再调用任何 HTTP 接口，否则 SPEED 和后续 INIT 消息均会被后端静默丢弃。

---

### 步骤 1：预设倍速为 3

#### 1.1 前端发起请求

**请求方式：** GET
**请求路径：** `/plan/speed`
**完整 URL：**
```
GET http://localhost:8080/plan/speed?dbName=plandeduce&speed=3&sessionId=s2
```

**参数说明：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `dbName` | String | 是 | 业务库名 |
| `speed` | Integer | 是 | 倍速值，0 等价于暂停，正整数为播放倍速 |
| `sessionId` | String | 否 | 会话标识 |
| `uuid` | String | 否 | 可选请求 ID，当前版本后端未使用 |

#### 1.2 后端处理链路

```
PlanDeduceController.speed()
  ├─ taskManager.getOrCreate(dbName, "s2")
  │    └─ 若不存在则 new ScenarioTask(...)，默认 speed=1, running=false, initialized=false
  └─ task.setSpeed(3)
       ├─ value = 3，wasRunning = running.get() = false
       ├─ speed.set(3)
       ├─ value != 0，且 wasRunning == false → running 保持 false（不改变）
       └─ pushStatus("SPEED", calculateNearestFullTime(0), "倍速已设置为 3")
              // currentTime=0，fullTime=0
```

**关键逻辑（`setSpeed` 内）：**
- 若原先 `running=true` → 改速后保持 `running=true`（播放中变速不停止）
- 若原先 `running=false` → 改速后保持 `running=false`（预设倍速不触发播放）
- 若 `speed=0` → 强制 `running=false`（0 倍速等价暂停）

#### 1.3 HTTP 响应

- **状态码：** `200`
- **响应体：** 空

#### 1.4 后端推送 SPEED 消息

```json
{
  "type": "SPEED",
  "dbName": "plandeduce",
  "sessionId": "s2",
  "currentTime": 0,
  "fullTime": 0,
  "speed": 3,
  "running": false,
  "message": "倍速已设置为 3"
}
```

**注意：**
- SPEED 是状态类消息，**不携带** `fullData`/`incrementalData`/`data`
- `running=false` 表示此时任务尚未开始播放
- `speed=3` 表示倍速已预设成功

#### 1.5 前端接收处理

```javascript
case 'SPEED':
  state.speed   = msg.speed;    // 3
  state.running = msg.running;  // false，尚未开始播放
  // 更新倍速显示（如 "3x"）
  // 不改变播放/暂停按钮状态
  updateSpeedDisplay(msg.speed);
  break;
```

### 步骤 2：触发开始播放（首次，含自动初始化）

#### 2.1 前端发起请求

**请求方式：** GET
**请求路径：** `/plan/startOrStop`
**完整 URL：**
```
GET http://localhost:8080/plan/startOrStop?dbName=plandeduce&flag=1&sessionId=s2
```

**参数说明：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `dbName` | String | 是 | 业务库名 |
| `flag` | Integer | 是 | 1=开始 |
| `sessionId` | String | 否 | 会话标识 |

#### 2.2 后端处理链路（关键路径，场景二特有）

```
PlanDeduceController.startOrStop()
  └─ task.startOrStop(flag=1)
       └─ task.resume()
            ├─ speed.get() = 3 > 0 → 不需要兜底
            ├─ initialized.get() == false   ← 场景二特有分支
            │    ├─ initializeRuntimeState()
            │    │    ├─ preloadFullSnapshots(dbName, 600)
            │    │    ├─ maxSimTime.set(queryMaxSimTime(dbName))
            │    │    └─ initialized.set(true)
            │    ├─ running.set(true)
            │    ├─ pushCurrentStateSnapshot("INIT")   ← 推 INIT，不推 START
            │    └─ ensureWorkerScheduled()             ← 注册 tick
            └─ [return，不走下面的 START 分支]
```

**与场景一 `start()` 的差异对比：**

| 对比项 | 场景一（`sendPlanDeduce`） | 场景二（`startOrStop(flag=1)`） |
|--------|--------------------------|-------------------------------|
| 入口方法 | `task.start(skip, null)` | `task.resume()` |
| 初始化检查 | 无条件执行 | 检查 `initialized` |
| 推送消息 | INIT | INIT（因 `initialized=false`） |
| 倍速 | 使用默认 speed=1 | 使用预设的 speed=3 |

#### 2.3 HTTP 响应

- **状态码：** `200`
- **响应体：** 空

#### 2.4 后端推送 INIT 消息（速度体现在 speed=3）

```json
{
  "type": "INIT",
  "dbName": "plandeduce",
  "sessionId": "s2",
  "currentTime": 0,
  "fullTime": 0,
  "speed": 3,
  "running": true,
  "fullData": [
    {
      "roomObjectId": 1,
      "objCode": "RED_ROOK",
      "side": "RED",
      "objName": "Red Rook Battalion",
      "currentPos": 100,
      "nextPos": 101,
      "direction": 0,
      "currentSpeed": 800,
      "visible": 1,
      "simTime": 0,
      "targetId": 1,
      "sourceType": "FULL"
    }
  ],
  "incrementalData": [],
  "data": [ { "simTime": 0, "sourceType": "FULL", ... } ],
  "message": "当前时间 0 秒，返回第 0 秒全量数据"
}
```

**与场景一 INIT 的区别：**
- `speed=3`（已生效为 3 倍速）
- 其余字段结构相同

#### 2.5 后端推送 PLAY 消息（3 倍速，每次步进 3 秒）

tick 内计算：`next = min(now + speed, maxSimTime) = min(0 + 3, 1200) = 3`

**PLAY 消息（t=3，增量区间 (0, 3]）：**
```json
{
  "type": "PLAY",
  "dbName": "plandeduce",
  "sessionId": "s2",
  "currentTime": 3,
  "fullTime": 0,
  "speed": 3,
  "running": true,
  "fullData": [],
  "incrementalData": [
    {
      "roomObjectId": 1,
      "objCode": "RED_ROOK",
      "side": "RED",
      "objName": "Red Rook Battalion",
      "currentPos": 103,
      "nextPos": 104,
      "direction": 6,
      "currentSpeed": 803,
      "visible": 1,
      "simTime": 3,
      "targetId": 31,
      "sourceType": "INCREMENT"
    }
  ],
  "data": [ { "simTime": 3, "sourceType": "INCREMENT", ... } ],
  "message": "当前时间 3 秒，返回第 1-3 秒增量数据"
}
```

**3 倍速下的增量语义：**
- `incrementalData` 是 `queryIncrementalData(dbName, 0, 3)` 的结果
- 查询 `SIM_TIME > 0 AND SIM_TIME <= 3` 的全部变化记录
- 包含 1、2、3 秒内所有棋子的历史变化条目（不去重）
- `message` 中体现为 `"第 1-3 秒增量数据"`

**全量间隔点 PLAY（t=600，因为 600 % 600 == 0）：**
```json
{
  "type": "PLAY",
  "dbName": "plandeduce",
  "sessionId": "s2",
  "currentTime": 600,
  "fullTime": 600,
  "speed": 3,
  "running": true,
  "fullData": [ { "simTime": 600, "sourceType": "FULL", ... } ],
  "incrementalData": [],
  "data": [ { "simTime": 600, "sourceType": "FULL", ... } ],
  "message": "当前时间 600 秒，返回第 600 秒全量数据"
}
```

#### 2.6 前端接收处理

```javascript
case 'INIT':
  state.initialized = true;
  state.running     = msg.running;      // true
  state.speed       = msg.speed;        // 3 ← 3 倍速已在 INIT 中确认
  state.currentTime = msg.currentTime;  // 0
  state.fullTime    = msg.fullTime;     // 0
  renderMap(msg.data);
  updateSpeedDisplay(msg.speed);        // 显示 "3x"
  break;

case 'PLAY':
  state.currentTime = msg.currentTime;  // 3, 6, 9, ...
  state.fullTime    = msg.fullTime;
  state.speed       = msg.speed;        // 保持 3
  renderMap(msg.data);
  updateProgressBar(msg.currentTime);
  // 注意：3 倍速时进度条每秒跳 3 格，不要插值补帧
  break;
```

**前端特别注意：**
- 3 倍速下 `currentTime` 序列为 0→3→6→9→…，不要自行补插 1、2 秒的本地帧
- 倍速来自 WS 消息的 `speed` 字段，不要在 HTTP 200 后就假定倍速已切换

```
GET http://localhost:8080/plan/startOrStop?dbName=plandeduce&flag=0&sessionId=s2
```

**预期 PAUSE 消息：**
```json

  "type": "PAUSE",
  "dbName": "plandeduce",

  "speed": 3,
  "running": false,
  "message": "已暂停"

```

**关键点：** `speed` 仍保持 3，暂停不重置倍速。

## 三、场景二完整消息时序

```
[WebSocket 建连]

调 speed=3
  → WS: SPEED（speed=3，running=false）

调 startOrStop(flag=1)
  → WS: INIT （speed=3，running=true，fullData 有值）  ← 首次开始是 INIT，不是 START
  → WS: PLAY (t=3)
  → WS: PLAY (t=6)
  → WS: PLAY (t=9)

调 startOrStop(flag=0)
  → WS: PAUSE（speed=3，running=false）

调 startOrStop(flag=1)
  → WS: START（speed=3，running=true）  ← 恢复后是 START，不是 INIT
  → WS: PLAY (t=12)
  → WS: PLAY (t=15)
  ...
  → WS: PLAY (t=1200)  ← 全量快照
  → WS: PAUSE
  → WS: FINISH
```

## 四、重要注意事项

### 前端

| 项目 | 说明 |
|------|------|
| 首次开始消息类型 | 场景二首次开始收到的是 **INIT**（不是 START），因为 `initialized=false` 时 `resume()` 走初始化分支 |
| 速度确认时机 | 收到 SPEED 消息才表示倍速预设成功，不能仅凭 HTTP 200 假设 |
| INIT 中的速度 | INIT 消息里的 `speed=3` 表示任务已以 3 倍速启动，前端应同步更新速度显示 |
| 进度条步进 | 3 倍速下 `currentTime` 以 3 为步长跳变，不要插值 |
| START vs INIT 区分 | `initialized=false` → 首次开始 → 收 INIT；`initialized=true` → 暂停后恢复 → 收 START |
| 恢复后速度不重置 | START 消息中 `speed=3`，前端不得将速度重置为 1 |
| 操作串行化建议 | 发出 speed 命令后等 SPEED 消息，再发 startOrStop；不要并发发送控制命令 |

### 后端

| 项目 | 说明 |
|------|------|
| `getOrCreate` 的副作用 | `speed` 接口调用 `getOrCreate` 会创建"半初始化任务"（`initialized=false`，有 speed，无 worker），属预期行为 |
| 倍速预设后再调 `sendPlanDeduce` | 若前端先调 `speed=3` 再调 `sendPlanDeduce`，后者会在 `task.start()` 里沿用已有的 `speed=3`，不会重置 |
| `resume()` 的双分支 | `initialized=false` 走初始化+INIT 分支；`initialized=true` 走 START 分支；确保首次启动行为与场景一一致 |
| 3 倍速增量数据量 | 每帧 `incrementalData` 包含 3 秒的变化记录（区间 `(prevTime, prevTime+3]`），数据量约为 1 倍速的 3 倍 |
| Worker 注册幂等 | `ensureWorkerScheduled()` 检查 `future == null || isCancelled() || isDone()`，重复调用不会创建多个 worker |
