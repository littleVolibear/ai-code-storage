当前真实行为：
1. 先调 speed=3 只是预设倍速，不会自动开始播放。
2. 如果任务还没初始化，第一次 startOrStop(flag=1) 会自动初始化。
3. 因为是首次开始，所以会先收到 INIT，不是 START。
4. INIT 之后按 3 倍速推 PLAY，时间一般是 0 -> 3 -> 6 -> 9。
5. 暂停后再次开始，才会收到 START，然后继续 PLAY。

协议注意：
1. 这个场景可以不先调 sendPlanDeduce。
2. data 是主数据，eventData 是事件数据。
3. fullData / incrementalData 当前固定为空数组。
