场景一当前已验证通过，对应测试：
PlanDeduceIntegrationTest.shouldStartPauseAndResumeAtOneX

当前真实行为：
1. sendPlanDeduce 只初始化，不发 WS 数据。
2. sendPlanDeduce 返回 InitProgressResponse，包含 dbName、sessionId、startTime、endTime。
3. 第一次 startOrStop(flag=1) 才会先发 INIT，再发 PLAY。
4. pause 时发 PAUSE。
5. 恢复播放时发 START，然后继续发 PLAY。
6. 到终点时顺序是 PLAY -> PAUSE -> FINISH。

协议注意：
1. 前端主消费 data 和 eventData。
2. fullData / incrementalData 现在固定发空数组。
3. 时间字段要区分 realTime 和 deduceTime，不再使用 currentTime。
4. HTTP 200 只代表命令收到了，前端状态以后端 WebSocket 为准。
