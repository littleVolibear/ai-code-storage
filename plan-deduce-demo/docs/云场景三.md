# 云场景三：1 倍速播到 5 秒 -> 暂停 -> 改 3 倍速 -> 自动恢复播放

## 结论

当前代码支持这个场景，对应测试是 `PlanDeduceIntegrationTest.shouldSupportDocumentedScenarioThreeFlow`。

以现在的代码为准，这个场景的关键点是：

1. 先通过 `sendPlanDeduce` 初始化。
2. 第一次 `startOrStop?flag=1` 开始播放。
3. 播到 `realTime >= 5` 且 `deduceTime = 5` 后暂停。
4. 暂停状态下调 `speed=3` 会先推 `SPEED`，然后自动推 `START`。
5. 恢复后的下一帧是 `realTime=6, deduceTime=8`，说明 3 倍速已经生效。

这和旧文档里“暂停后调速不会自动恢复，还要再调一次开始接口”已经不同。

## 推荐时序

```text
1. 建立 WebSocket
2. GET /plan/sendPlanDeduce?dbName=1&skip=0&sessionId=s3
3. HTTP 返回 InitProgressResponse
4. GET /plan/startOrStop?dbName=1&flag=1&sessionId=s3
5. WS: INIT(realTime=0, deduceTime=0, speed=1)
6. WS: PLAY(realTime=1, deduceTime=1)
7. WS: PLAY(realTime=2, deduceTime=2)
8. WS: PLAY(realTime=3, deduceTime=3)
9. WS: PLAY(realTime=4, deduceTime=4)
10. WS: PLAY(realTime=5, deduceTime=5)
11. GET /plan/startOrStop?flag=0
12. WS: PAUSE(realTime=5, deduceTime=5, speed=1, running=false)
13. GET /plan/speed?speed=3
14. WS: SPEED(realTime=5, deduceTime=5, speed=3, running=false)
15. WS: START(realTime=5, deduceTime=5, speed=3, running=true)
16. WS: PLAY(realTime=6, deduceTime=8)
17. 后续 deduceTime 按 11、14、17 ... 推进
```

## 前端判断暂停时机

正确做法：

- 以前端收到的 `PLAY.realTime >= 5` 且 `PLAY.deduceTime = 5` 作为“已经播放到第 5 秒”的依据。

不要这样做：

- 本地自己计时 5 秒后直接认为可以暂停。

原因：

- 当前播放推进完全以后端 `PLAY` 消息为准。
- `realTime` 和 `deduceTime` 是分开的，不能只看浏览器本地计时。

## 关键消息语义

### 初始化

`sendPlanDeduce` 只返回：

```json
{
  "dbName": "1",
  "sessionId": "s3",
  "startTime": "2026-01-01 00:00:00",
  "endTime": 1200
}
```

不会直接推 `INIT`。

### 第一次开始

第一次 `startOrStop?flag=1` 推 `INIT`，随后推 `PLAY`。

### 暂停后调速

`speed=3` 会先更新速度，再自动恢复：

```json
{
  "type": "SPEED",
  "realTime": 5,
  "deduceTime": 5,
  "speed": 3,
  "running": false
}
```

随后会再收到：

```json
{
  "type": "START",
  "realTime": 5,
  "deduceTime": 5,
  "speed": 3,
  "running": true
}
```

下一帧：

```json
{
  "type": "PLAY",
  "realTime": 6,
  "deduceTime": 8,
  "speed": 3,
  "running": true
}
```

## 数据字段约定

1. 地图渲染使用 `data`。
2. 事件渲染使用 `eventData`。
3. `fullData`、`incrementalData` 当前固定为空数组，不做业务消费。
4. `fullTime` 表示最近全量秒点，业务时间要看 `deduceTime`。
