# 云场景三：1 倍速播放到 5 秒 → 暂停 → 改 3 倍速 → 继续播放 → 结束

> 本文档面向前后端联合协作，覆盖后端如何发送数据、前端如何接收处理、双方各自注意事项，以及当前实现进度。
> 服务地址：`http://localhost:8080`
> WebSocket 地址：`ws://localhost:8080/ws/planDeduce`

---

## 一、场景概述

先以默认 1 倍速正常播放，当业务时间到达 5 秒后暂停，在暂停状态下调整倍速为 3，再恢复播放，以 3 倍速一直推进至结束。

**场景三与场景二的核心区别：**
- 场景二：倍速切换发生在**初始化之前**（未 start 时预设）
- 场景三：倍速切换发生在**已初始化且已暂停之后**（更稳定，不涉及"worker 未启动"问题）

**完整交互顺序：**

```
前端                               后端
 |                                  |
 |-- 建立 WebSocket 连接 ---------->|  登记 sessionId 映射
 |                                  |
 |-- GET /plan/sendPlanDeduce ----->|  task.start(skip=0, null)
 |                                  |  initialized=true，speed=1，running=true
 |<-- WS: INIT -------------------|  speed=1，running=true
 |<-- WS: PLAY (t=1) -------------|
 |<-- WS: PLAY (t=2) -------------|
 |<-- WS: PLAY (t=3) -------------|
 |<-- WS: PLAY (t=4) -------------|
 |<-- WS: PLAY (t=5) -------------|  ← 前端确认 currentTime >= 5
 |                                  |
 |-- GET /plan/startOrStop?flag=0 ->|  task.pause()
 |                                  |  running=false
 |<-- WS: PAUSE -------------------|  speed=1，running=false，currentTime=5
 |                                  |
 |-- GET /plan/speed?speed=3 ------>|  task.setSpeed(3)
 |                                  |  wasRunning=false，speed=3，running 不变（false）
 |<-- WS: SPEED -------------------|  speed=3，running=false，currentTime=5
 |                                  |
 |-- GET /plan/startOrStop?flag=1 ->|  task.resume()
 |                                  |  initialized=true → running=true，pushStatus("START")
 |<-- WS: START -------------------|  speed=3，running=true，currentTime=5
 |<-- WS: PLAY (t=8) -------------|  tick#1：step=3，next=min(5+3,1200)=8
 |<-- WS: PLAY (t=11) ------------|
 |        ...                       |
 |<-- WS: PLAY (t=1200) ----------|
 |<-- WS: PAUSE -------------------|
 |<-- WS: FINISH ------------------|
```

### 步骤 2：在 5 秒处暂停

#### 2.1 时机判断（重要）

**正确做法：** 前端收到 `PLAY.currentTime >= 5` 后，用户点击暂停，再发起 HTTP 请求。
**错误做法：** 前端本地定时 5 秒后自动触发暂停，忽略实际推送时刻。

#### 2.2 前端发起请求

**请求方式：** GET
**请求路径：** `/plan/startOrStop`
**完整 URL：**
```
GET http://localhost:8080/plan/startOrStop?dbName=plandeduce&flag=0&sessionId=s3
```

#### 2.3 后端处理链路

```
PlanDeduceController.startOrStop()
  └─ task.startOrStop(flag=0)
       └─ task.pause()
            ├─ running.set(false)
            ├─ speed.compareAndSet(0, 1)    // speed=1 不触发，保持 1
            └─ pushStatus("PAUSE", calculateNearestFullTime(5), "已暂停")
                  // calculateNearestFullTime(5) = (5/600)*600 = 0
```

#### 2.4 HTTP 响应

- **状态码：** `200`
- **响应体：** 空

#### 2.5 后端推送 PAUSE 消息

```json
{
  "type": "PAUSE",
  "dbName": "plandeduce",
  "sessionId": "s3",
  "currentTime": 5,
  "fullTime": 0,
  "speed": 1,
  "running": false,
  "message": "已暂停"
}
```

**数据说明：**
- `currentTime=5`：暂停时刻
- `speed=1`：暂停不改变速度
- `running=false`：任务停止推进
- 不携带 `fullData`/`incrementalData`/`data`

#### 2.6 前端接收处理

```javascript
case 'PAUSE':
  state.running     = false;
  state.currentTime = msg.currentTime;  // 5
  state.speed       = msg.speed;        // 1（暂停时记录下当前速度）
  // 切换 UI 到暂停态
  // 此时允许用户调整倍速
  updateUI('paused');
  enableSpeedControls();
  break;
```

### 步骤 3：在暂停状态下将倍速改为 3

#### 3.1 前端发起请求

**必须在收到 PAUSE 消息后才发起此请求，确保任务已经进入暂停态。**

**请求方式：** GET
**请求路径：** `/plan/speed`
**完整 URL：**
```
GET http://localhost:8080/plan/speed?dbName=plandeduce&speed=3&sessionId=s3
```

**参数说明：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `dbName` | String | 是 | 业务库名 |
| `speed` | Integer | 是 | 新倍速值，此处为 3 |
| `sessionId` | String | 否 | 会话标识 |

#### 3.2 后端处理链路

```
PlanDeduceController.speed()
  └─ task.setSpeed(3)
       ├─ value = 3
       ├─ wasRunning = running.get() = false   ← 当前处于暂停态
       ├─ speed.set(3)
       ├─ value != 0，且 wasRunning == false
       │    └─ running 保持 false（不自动恢复播放）  ← 场景三的关键规则
       └─ pushStatus("SPEED", calculateNearestFullTime(5), "倍速已设置为 3")
              // currentTime=5，fullTime=0
```

**关键规则：暂停后改倍速，不会自动恢复播放。**

`setSpeed` 完整逻辑：
```java
public void setSpeed(Integer newSpeed) {
    int value = newSpeed == null ? 1 : newSpeed;
    boolean wasRunning = running.get();          // false（暂停中）
    speed.set(Math.max(value, 0));               // speed = 3
    if (value == 0) {
        running.set(false);                      // speed=0 → 强制暂停
    } else if (wasRunning) {
        running.set(true);                       // 播放中改速 → 保持播放
    }
    // wasRunning=false 且 value≠0 → 不执行任何 running 变更
    pushStatus("SPEED", ...);
}
```

#### 3.3 HTTP 响应

- **状态码：** `200`
- **响应体：** 空

#### 3.4 后端推送 SPEED 消息

```json
{
  "type": "SPEED",
  "dbName": "plandeduce",
  "sessionId": "s3",
  "currentTime": 5,
  "fullTime": 0,
  "speed": 3,
  "running": false,
  "message": "倍速已设置为 3"
}
```

**数据说明：**
- `speed=3`：倍速已成功切换
- `running=false`：**仍处于暂停态**，改倍速不恢复播放
- `currentTime=5`：时间未推进

#### 3.5 前端接收处理

```javascript
case 'SPEED':
  state.speed   = msg.speed;    // 3
  state.running = msg.running;  // false（仍然暂停）
  // 更新速度显示
  updateSpeedDisplay(msg.speed);  // 显示 "3x"
  // 此时 UI 状态：暂停中 + 倍速已改为 3
  // 必须等 SPEED 消息到达后，才允许下一步"点击继续播放"
  enableResumeButton();
  break;
```

**前端必须注意：**
- 收到 SPEED 消息后 `running` 仍为 `false`
- 此时必须再调一次 `startOrStop(flag=1)` 才能恢复播放
- **不能**收到 SPEED 就认为已经在播放了

#### 4.6 前端接收处理

```javascript
case 'START':
  state.running     = true;
  state.currentTime = msg.currentTime;  // 5（暂停时刻，不变）
  state.speed       = msg.speed;        // 3（已切换）
  updateUI('playing');
  // 注意：此时进度条仍显示 5 秒，不要跳动，等下一条 PLAY

case 'PLAY':
  const prev = state.currentTime;       // 5
  state.currentTime = msg.currentTime;  // 8（直接跳 3 秒）
  state.speed       = msg.speed;        // 3
```

- `currentTime` 从 5 直接跳到 8，是 3 倍速的正常行为

- 所有数据更新必须以 `PLAY.data` 为准

## 三、场景三完整消息时序

```
[WebSocket 建连]

调 sendPlanDeduce
  → WS: INIT（speed=1，running=true，fullData 有值）

  → WS: PLAY (t=1，speed=1)
  → WS: PLAY (t=2，speed=1)
  → WS: PLAY (t=3，speed=1)
  → WS: PLAY (t=4，speed=1)
  → WS: PLAY (t=5，speed=1)    ← 前端确认 currentTime=5，允许暂停

调 startOrStop(flag=0)
  → WS: PAUSE（speed=1，running=false，currentTime=5）

调 speed=3
  → WS: SPEED（speed=3，running=false，currentTime=5）  ← 仍暂停，未恢复

调 startOrStop(flag=1)
  → WS: START（speed=3，running=true，currentTime=5）
  → WS: PLAY (t=8，speed=3)   ← 从 5 秒以 3 为步长跳到 8
  → WS: PLAY (t=11，speed=3)
  → WS: PLAY (t=14，speed=3)
  ...
  → WS: PLAY (t=1200，speed=3，全量快照)
  → WS: PAUSE（speed=3，running=false）
  → WS: FINISH（speed=3，running=false）
```

## 四、场景三与其他场景的对比

| 对比项 | 场景一 | 场景二 | 场景三 |
|--------|--------|--------|--------|
| 倍速切换时机 | 无切换（全程 1 倍速） | 初始化前预设 3 倍速 | 暂停后切换为 3 倍速 |
| 首次开始消息 | INIT（via `sendPlanDeduce`） | INIT（via `resume()` 自动初始化） | INIT（via `sendPlanDeduce`） |
| 倍速切换触发消息 | 无 | SPEED（`running=false`，初始化前） | SPEED（`running=false`，暂停后） |
| 恢复播放消息 | START | START（第二次及以后） | START |
| 步长变化时刻 | 全程步长=1 | 全程步长=3 | 5 秒前步长=1，5 秒后步长=3 |
| Worker 启动时机 | `start()` 中 `ensureWorkerScheduled` | `resume()` 中 `ensureWorkerScheduled` | `start()` 中，后续 `resume()` 不重启 |
| 初始化来源 | `task.start()` | `task.resume()` 内自动初始化 | `task.start()` |

## 五、数据字段详细说明

### `RoomObjectHis` 前端核心字段

下表列出了前端在渲染地图时最常使用的字段，以及在场景三中的典型值变化。

| 字段 | Java 类型 | 说明 | 1 倍速阶段（t=1~5） | 3 倍速阶段（t=8~1200） |
|------|-----------|------|---------------------|------------------------|
| `roomObjectId` | Integer | 棋子唯一 ID（主键） | 不变 | 不变 |
| `objCode` | String | 棋子编号（图标键） | 不变 | 不变 |
| `side` | String | 阵营（RED/BLUE） | 不变 | 不变 |
| `objName` | String | 棋子名称 | 不变 | 不变 |
| `currentPos` | Integer | 当前位置格子 ID | 每秒变化 | 每 3 秒变化 |
| `nextPos` | Integer | 下一步位置 | 每秒变化 | 每 3 秒变化 |
| `direction` | Integer | 朝向（0-360 度） | 每秒变化 | 每 3 秒变化 |
| `currentSpeed` | Integer | 机动速度（ms/格） | 每秒可能变化 | 每 3 秒可能变化 |
| `moving` | Integer | 是否机动中（0/1） | 可变 | 可变 |
| `visible` | Integer | 是否可见（0/1） | 可变 | 可变 |
| `simTime` | Integer | 数据业务时刻 | 1,2,3,4,5 | 8,11,14,... |
| `targetId` | Long | 打击目标 ID | 可变 | 可变 |
| `sourceType` | String | FULL / INCREMENT | INCREMENT（t=1~4）；全量点为 FULL | 同左 |

### `data` 字段的使用建议

后端在 `PlanDeducePush.pushSnapshot` 中统一将 `fullData + incrementalData` 合并为 `data`：
```java
List<RoomObjectHis> mergedData = new ArrayList<>();
mergedData.addAll(fullData);
mergedData.addAll(incrementalData);
message.setData(mergedData);
```

**前端使用建议：**
- **简单场景**：直接消费 `data` 字段，按 `roomObjectId` 更新地图上的对应棋子
- **精细场景**：分别消费 `fullData` 和 `incrementalData`，自行处理快照合并逻辑
- 增量帧中 `data` 只包含**本帧变化的棋子**，未变化的棋子不会出现，前端应保留上一帧的状态

## 六、重要注意事项

### 前端

| 项目 | 说明 |
|------|------|
| 暂停时机判断 | 只能以 `PLAY.currentTime >= 5` 为准，禁止本地计时 |
| 调速时机 | 必须在收到 PAUSE 后再调 `speed=3`，确保任务已暂停 |
| 调速后不自动播放 | 收到 SPEED 消息时 `running=false`，必须再调 `startOrStop(flag=1)` |
| 操作严格串行 | 正确顺序：① 等 PAUSE → ② 调 speed → ③ 等 SPEED → ④ 调 startOrStop(flag=1) |
| 进度条跳跃 | 恢复后 `currentTime` 从 5 直接到 8，此为正常行为，禁止补帧 |
| START 消息语义 | `currentTime=5` 表示恢复命令成功，进度条不动，等第一条 PLAY(t=8) 才更新 |
| 增量数据不完整 | `incrementalData` 只包含变化的棋子，未变化棋子不在其中，前端需自行合并历史状态 |

### 后端

| 项目 | 说明 |
|------|------|
| `setSpeed` 在暂停时的行为 | `wasRunning=false && value≠0` → `running` 不变，speed 更新，**不自动恢复** |
| `setSpeed` 在播放时的行为 | `wasRunning=true && value≠0` → `running` 保持 true，速度立即生效于下一 tick |
| `setSpeed(0)` 等价暂停 | 强制 `running=false`，下次 `resume()` 时 speed 会被兜底为 1 |
| tick 不因暂停停止 | `pause()` 只将 `running` 置 false，worker 仍运行，tick 在入口返回 |
| 恢复后第一帧步长 | `next = min(5 + 3, 1200) = 8`，增量区间 `(5, 8]`，包含 6、7、8 秒变化 |
| 增量数据排序 | `queryIncrementalData` 按 `simTime ASC, roomObjectId ASC, targetId ASC` 排序 |
| 全量间隔点规则 | `1200 % 600 == 0`，最后一帧推全量而非增量 |
