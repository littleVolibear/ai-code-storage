# 后端自测场景说明

> 适用版本：plan-deduce-demo 当前代码
> 服务默认地址：`http://localhost:8080`
> WebSocket 地址：`ws://localhost:8080/ws/planDeduce`

## 一、基础约定

### 1.1 接口总览

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 初始化任务 | GET | `/plan/sendPlanDeduce` | 只初始化并返回进度条范围 |
| 开始 / 暂停 | GET | `/plan/startOrStop` | `flag=1` 开始或恢复，`flag=0` 暂停 |
| 设置倍速 | GET | `/plan/speed` | 传入整数倍速值 |
| 跳转秒点 | GET | `/plan/skip` | 跳转到指定业务秒 |
| 修改全量间隔 | GET | `/plan/fullSaveInterval` | 修改快照间隔秒数 |
| 销毁任务 | GET | `/plan/destroy` | 释放后端资源 |

### 1.2 HTTP 响应规则

- 正常：HTTP 200
- `sendPlanDeduce`：返回 `InitProgressResponse`
- 其他控制接口：通常无响应体
- 异常：HTTP 400，返回错误说明

### 1.3 WebSocket 消息关键字段

| 字段 | 说明 |
|------|------|
| `type` | INIT / PLAY / PAUSE / START / FINISH / SPEED / SKIP / INTERVAL / DESTROY / ERROR |
| `dbName` | 兼容字段，原样回传 |
| `sessionId` | 当前任务唯一标识 |
| `realTime` | 真实 tick 时间 |
| `deduceTime` | 推演业务时间 |
| `fullTime` | 最近全量快照秒点 |
| `speed` | 当前倍速 |
| `running` | 是否处于播放态 |
| `data` | 前端主消费棋子数据 |
| `eventData` | 前端主消费事件数据 |
| `message` | 说明文案 |

补充说明：

1. 当前协议已不再使用 `currentTime`。
2. `fullData` / `incrementalData` 当前固定为空数组，只作兼容保留。
3. 当前任务只按 `sessionId` 隔离。

## 二、推荐自测方式

### 方式 A：curl + wscat

```bash
wscat -c "ws://localhost:8080/ws/planDeduce?sessionId=s1"
curl "http://localhost:8080/plan/sendPlanDeduce?dbName=plandeduce&skip=0&sessionId=s1"
```

### 方式 B：项目内置静态页

```text
http://localhost:8080/index.html
```

### 方式 C：Maven 集成测试

```bash
mvn test -Dtest=PlanDeduceIntegrationTest
```

## 三、场景一：初始化 -> 开始 -> 暂停 -> 恢复

步骤：

```bash
curl "http://localhost:8080/plan/sendPlanDeduce?dbName=plandeduce&skip=0&sessionId=s1"
curl "http://localhost:8080/plan/startOrStop?dbName=plandeduce&flag=1&sessionId=s1"
curl "http://localhost:8080/plan/startOrStop?dbName=plandeduce&flag=0&sessionId=s1"
curl "http://localhost:8080/plan/startOrStop?dbName=plandeduce&flag=1&sessionId=s1"
```

预期：

1. 初始化 HTTP 返回：

```json
{
  "dbName": "plandeduce",
  "sessionId": "s1",
  "startTime": "2026-01-01 00:00:00",
  "endTime": 1200
}
```

2. 首次开始后收到 `INIT`
3. 后续收到 `PLAY`
4. 暂停时收到 `PAUSE`
5. 恢复时收到 `START`

## 四、场景二：播放中切到 3 倍速

步骤：

```bash
curl "http://localhost:8080/plan/speed?dbName=plandeduce&speed=3&sessionId=s1"
```

预期：

1. 收到 `SPEED`
2. 如果原本正在播放，后续 `PLAY` 继续出现
3. `PLAY.realTime` 每次加 `1`
4. `PLAY.deduceTime` 每次加 `3`

## 五、场景三：暂停后调成 3 倍速

步骤：

```bash
curl "http://localhost:8080/plan/startOrStop?dbName=plandeduce&flag=0&sessionId=s1"
curl "http://localhost:8080/plan/speed?dbName=plandeduce&speed=3&sessionId=s1"
```

预期：

1. 先收到 `PAUSE`
2. 再收到 `SPEED`
3. 因为当前代码会自动恢复，所以还会收到 `START`
4. 后续重新收到 `PLAY`

注意：

- 这和旧文档里“暂停后调速不会自动恢复”不同

## 六、场景四：跳到指定秒点

步骤：

```bash
curl "http://localhost:8080/plan/skip?dbName=plandeduce&skip=13&sessionId=s1"
```

预期：

1. 收到 `SKIP`
2. `realTime=13`
3. `deduceTime=13`
4. `fullTime=10`

注意：

- 主数据看 `data`
- 事件看 `eventData`
- 不要再要求 `fullData` / `incrementalData` 对外有真实数据

## 七、场景五：暂停后 skip

步骤：

```bash
curl "http://localhost:8080/plan/startOrStop?dbName=plandeduce&flag=0&sessionId=s1"
curl "http://localhost:8080/plan/skip?dbName=plandeduce&skip=33&sessionId=s1"
```

预期：

1. 先收到 `PAUSE`
2. `skip` 后会自动恢复
3. 常见顺序是 `START -> SKIP -> PLAY`

## 八、场景六：播放结束

步骤：

```bash
curl "http://localhost:8080/plan/sendPlanDeduce?dbName=plandeduce&skip=1198&sessionId=s1"
curl "http://localhost:8080/plan/startOrStop?dbName=plandeduce&flag=1&sessionId=s1"
curl "http://localhost:8080/plan/speed?dbName=plandeduce&speed=3&sessionId=s1"
```

预期：

1. 最后一条 `PLAY`：`realTime=1199, deduceTime=1200`
2. 随后 `PAUSE`
3. 再随后 `FINISH`

## 九、排查提示

如果联调现象和旧文档不一致，优先按下面口径判断当前代码：

1. `/plan/speed` 在 `speed>0 && running=false` 时会自动恢复
2. `/plan/skip` 在 `running=false` 时会自动恢复
3. 任务只按 `sessionId` 隔离
4. 对外时间字段是 `realTime` / `deduceTime`
