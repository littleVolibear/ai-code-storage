# 后端自测场景说明

> 适用版本：plan-deduce-demo
> 服务默认地址：`http://localhost:8080`
> WebSocket 地址：`ws://localhost:8080/ws/planDeduce`

---

## 一、基础约定

### 1.1 接口总览

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 初始化并开始播放 | GET | `/plan/sendPlanDeduce` | 第一次进入或销毁后重新开始 |
| 开始 / 暂停 | GET | `/plan/startOrStop` | flag=1 开始，flag=0 暂停 |
| 设置倍速 | GET | `/plan/speed` | 传入整数倍速值 |
| 跳转秒点 | GET | `/plan/skip` | 跳转到指定业务秒 |
| 修改全量间隔 | GET | `/plan/fullSaveInterval` | 修改快照间隔秒数 |
| 销毁任务 | GET | `/plan/destroy` | 释放后端资源 |

### 1.2 HTTP 响应规则

- **正常**：HTTP 200，响应体为空
- **异常**（如重复初始化、参数缺失）：HTTP 400，响应体如下：

```json
{
  "success": false,
  "message": "错误说明"
}
```

HTTP 层只表示"命令是否提交成功"，**不代表业务状态已切换**。

### 1.3 WebSocket 消息总格式

后端通过 WebSocket 推送 JSON 消息，字段说明如下：

| 字段 | 说明 |
|------|------|
| `type` | 消息类型：INIT / PLAY / PAUSE / START / FINISH / SPEED / SKIP / INTERVAL / DESTROY / ERROR |
| `dbName` | 当前任务所属数据库 |
| `sessionId` | 会话标识 |
| `currentTime` | 当前播放到的业务秒点 |
| `fullTime` | 当前快照对应的最近全量秒点 |
| `speed` | 当前倍速 |
| `running` | 是否处于播放态 |
| `fullData` | 全量快照数据（快照类消息） |
| `incrementalData` | 增量数据（快照类消息） |
| `data` | fullData + incrementalData 合并结果 |
| `message` | 说明文案 |

**快照类消息**（携带数据数组）：INIT、PLAY、SKIP、INTERVAL
**状态类消息**（不携带数据数组）：PAUSE、START、SPEED、FINISH、DESTROY、ERROR

## 二、内网隔离环境测试工具说明

在无法使用 APIFox 等外部工具的内网隔离环境下，推荐以下几种方式进行自测，**无需安装任何额外软件**：

### 方式 A：curl（推荐，全平台内置）

Linux / macOS 系统均内置 curl，可直接在终端发起 HTTP 请求。

```bash
# 示例：发起初始化请求
curl -v "http://localhost:8080/plan/sendPlanDeduce?dbName=plandeduce&skip=0&sessionId=s1"
```

- `-v` 参数可显示完整请求/响应头，方便确认状态码
- HTTP 200 且无响应体即为成功

### 方式 B：项目内置静态测试页（推荐，可实时观察 WebSocket 消息）

项目已内置前端测试页，启动服务后直接用浏览器访问：

```
http://localhost:8080/index.html
```

页面提供"连接 WebSocket"及常用控制按钮，WebSocket 收到的每一条推送消息会实时打印在页面上，**是观察 WS 推送最直观的方式**。

> 注意：该页面的 sessionId 固定为 s1，场景二/三自测时建议搭配浏览器控制台脚本修改参数，见方式 C。

### 方式 C：浏览器控制台（可自定义 sessionId 及任意参数）

打开浏览器（Chrome/Firefox/Edge），按 F12 打开 DevTools，切换到 **Console** 标签页，直接粘贴以下脚本：

```javascript
// 1. 建立 WebSocket 连接（根据场景修改 sessionId）
let ws = new WebSocket('ws://localhost:8080/ws/planDeduce?sessionId=s1');
ws.onopen = () => console.log('[WS] 已连接');
ws.onmessage = e => console.log('[WS] 收到：', JSON.parse(e.data));
ws.onclose = () => console.log('[WS] 已断开');

// 2. 发起 HTTP 请求（根据场景修改 URL）
function call(url) {
  fetch(url).then(r => {
    console.log('[HTTP]', r.status, url);
    return r.text();
  }).then(t => t && console.log('[HTTP Body]', t));
}

// 示例：
call('/plan/sendPlanDeduce?dbName=plandeduce&skip=0&sessionId=s1');
```

此方式可完整看到 HTTP 状态码和 WebSocket 推送内容，也可在 DevTools **Network** 标签页的 **WS** 子项中逐帧查看 WebSocket 消息。

### 方式 D：Maven 集成测试（自动化验证，无需手动操作）

项目已包含集成测试类 `PlanDeduceIntegrationTest.java`，覆盖典型场景链路。在终端执行：

```bash
mvn test
```

或者只运行集成测试：

```bash
mvn test -pl . -Dtest=PlanDeduceIntegrationTest
```

测试结果会输出到 `target/surefire-reports/`，通过即表示主要链路正常。

### 方式 E：wscat（需离线 Node.js 环境）

若内网机器已安装 Node.js，可离线安装 wscat 包后使用：

```bash
# 安装（仅首次，需要 npm 离线包）
npm install -g wscat

# 连接 WebSocket
wscat -c "ws://localhost:8080/ws/planDeduce?sessionId=s1"
# 连接成功后，终端会实时打印服务端推送的每一条 JSON 消息
```

## 三、场景一：播放 → 暂停 → 继续播放 → 结束

**场景描述**：以默认 1 倍速初始化播放，中途暂停一次，继续播放后等待任务自动结束。

---

### 步骤 1：初始化并开始播放

**请求方式**：GET
**请求路径**：`/plan/sendPlanDeduce`
**完整请求**：
```
GET http://localhost:8080/plan/sendPlanDeduce?dbName=plandeduce&skip=0&sessionId=s1
```

**curl 命令**：
```bash
curl -v "http://localhost:8080/plan/sendPlanDeduce?dbName=plandeduce&skip=0&sessionId=s1"
```

**预期 HTTP 响应**：
- 状态码：`200`
- 响应体：空

**预期 WebSocket 消息**（先收到 INIT，随后持续收到 PLAY）：

INIT 示例：
```json
{
  "type": "INIT",
  "dbName": "plandeduce",
  "sessionId": "s1",
  "currentTime": 0,
  "fullTime": 0,
  "speed": 1,
  "running": true,
  "fullData": [ { "roomObjectId": 1, "objCode": "RED_ROOK", ... } ],
  "incrementalData": [],
  "data": [ { "roomObjectId": 1, "objCode": "RED_ROOK", ... } ],
  "message": "当前时间 0 秒，返回第 0 秒全量数据"
}
```

PLAY 示例（每秒推送一次）：
```json
{
  "type": "PLAY",
  "dbName": "plandeduce",
  "sessionId": "s1",
  "currentTime": 1,
  "fullTime": 0,
  "speed": 1,
  "running": true,
  "fullData": [],
  "incrementalData": [ { "roomObjectId": 1, "simTime": 1, "sourceType": "INCREMENT", ... } ],
  "data": [ { "roomObjectId": 1, "simTime": 1, ... } ],
  "message": "当前时间 1 秒，返回第 1-1 秒增量数据"
}
```

**验证要点**：
- [ ] HTTP 返回 200，响应体为空
- [ ] WebSocket 收到 `type=INIT`，`running=true`，`speed=1`，`fullData` 不为空
- [ ] 随后持续收到 `type=PLAY`，`currentTime` 每次递增 1

- [ ] 结束时收到的最后三条消息顺序为：PLAY → PAUSE → FINISH

**预期 WebSocket 消息**（先收到 START，随后以 3 倍速继续推进 PLAY）：

START 示例：
```json
{
  "type": "START",
  "dbName": "plandeduce",
  "sessionId": "s3",
  "currentTime": 5,
  "speed": 3,
  "running": true,
  "message": "已开始"
}
```

恢复后第一条 PLAY（从 5 秒直接跳到 8 秒，步长 3）：
```json
{
  "type": "PLAY",
  "dbName": "plandeduce",
  "sessionId": "s3",
  "currentTime": 8,
  "speed": 3,
  "running": true,
  "fullData": [],
  "incrementalData": [ { "simTime": 8, "sourceType": "INCREMENT", ... } ],
  "data": [ { ... } ],
  "message": "当前时间 8 秒，返回第 6-8 秒增量数据"
}
```

**验证要点**：
- [ ] HTTP 返回 200
- [ ] WebSocket 收到 `type=START`，`speed=3`，`running=true`
- [ ] 随后 PLAY 中 `currentTime` 从 5 直接步进到 8（步长 3，符合 3 倍速），不从 0 重置
- [ ] PLAY 消息中 `speed=3`，全程保持 3 倍速

---

### 步骤 5：等待播放结束（无需发起请求）

与场景一步骤 4 相同，结束时按顺序收到：PLAY（`currentTime=1200`）→ PAUSE → FINISH。

**验证要点**：
- [ ] 整个恢复后的播放阶段，PLAY 的 `currentTime` 均以 3 为步长递增
- [ ] FINISH 消息中 `speed=3`，`running=false`，`currentTime=1200`

## 七、测试执行顺序建议

每次测试新场景前，建议先销毁旧任务（或更换不同 sessionId），避免残留状态干扰：

```bash
# 清理场景一
curl "http://localhost:8080/plan/destroy?dbName=plandeduce&sessionId=s1"

# 清理场景二
curl "http://localhost:8080/plan/destroy?dbName=plandeduce&sessionId=s2"

# 清理场景三
curl "http://localhost:8080/plan/destroy?dbName=plandeduce&sessionId=s3"
```

或直接重启服务，H2 内存数据库会自动重建测试数据：

```bash
mvn spring-boot:run
```
